1. 首页（Home Page）

    1.1 页面结构
    首页由三部分组成：
    - **顶部导航栏**：包含“我的12306”、“登录”、“注册”入口。
    - **常用功能快捷入口**：包含“首页”、“车票”等模块。
    - **车票查询表单**：包含出发地、到达地、出发日期、学生票/高铁勾选框及“查询”按钮。

    1.2 页面跳转逻辑

        1.2.1 用户点击顶部导航栏“我的12306”
        - 若用户**未登录** → 自动跳转至**登录页**。
        - 若用户**已登录** → 不发生跳转

        1.2.2 用户点击顶部导航栏“登录”或“注册”
        - 点击“登录” → 进入**登录页**。
        - 点击“注册” → 进入**注册页**。

        1.2.3 用户点击常用功能快捷入口“首页”按钮
        - 若当前在其他页面 → 返回首页。
        - 若当前已在首页 → 刷新当前页面。

        1.2.4 用户点击常用功能快捷入口“车票”按钮
        - 不发生跳转


2. 登录页（Login Page）

    2.1 页面结构
    登录页包含：
    - 用户名/邮箱/手机号输入框
    - 密码输入框
    - “立即登录”按钮
    - “注册12306账号”链接
    - “忘记密码？”链接（暂不实现）

    2.2 登录场景

        2.2.1 用户输入不完整信息
        Scenario: 未输入用户名或密码  
        **Given** 用户在登录页面  
        **When** 用户点击“立即登录”按钮但未填写完整信息  
        **Then** 系统弹出提示框 “请输入用户名和密码”  

        2.2.2 用户输入错误格式的用户名/密码  
        Scenario: 输入格式错误  
        **Given** 用户在登录页面  
        **When** 用户输入格式不符合要求的用户名或密码（如包含非法字符）  
        **Then** 系统提示 “格式错误！”  

        2.2.3 用户输入未注册账户  
        Scenario: 使用未注册账号登录  
        **Given** 用户在登录页面  
        **When** 用户输入未注册的用户名或手机号并点击“立即登录”  
        **Then** 系统提示 “该账号未注册，请先注册”  

        2.2.4 用户输入错误密码  
        Scenario: 使用错误密码登录  
        **Given** 用户已注册  
        **When** 用户输入错误的密码并点击“立即登录”  
        **Then** 系统提示 “密码错误，请重新输入”  

        2.2.5 用户登录成功  
        Scenario: 成功登录  
        **Given** 用户已注册  
        **When** 用户输入正确的用户名/手机号与密码并点击“立即登录”  
        **Then** 系统提示 “登录成功”  
        **And** 页面跳转至首页  
        **And** 顶部导航栏更新为 “您好，XXX / 退出”  


3. 注册页（Register Page）

    3.1 页面结构
    注册页包含以下输入项：
    - 用户名  
    - 登录密码  
    - 确认密码  
    - 证件类型  
    - 姓名  
    - 证件号码  
    - 优惠（待）类型  
    - 手机号码  
    - 邮箱（选填）  
    - “我已阅读并确认条款”勾选框  
    - “下一步”按钮

    3.2 注册流程与场景

        3.2.1 未填写手机号点击下一步  
        Scenario: 手机号为空  
        **Given** 用户在注册页面  
        **When** 用户未填写手机号并点击“下一步”  
        **Then** 系统弹出提示 “请输入手机号”  

        3.2.2 部分必填项未填写  
        Scenario: 信息不完整  
        **Given** 用户在注册页面  
        **When** 用户仅填写部分必填项并点击“下一步”  
        **Then** 未填写项下方显示 “❌请输入XXX”  

        3.2.3 输入已注册的证件号码  
        Scenario: 重复注册  
        **Given** 某证件号码已存在于系统  
        **When** 用户填写该证件号码并点击“下一步”  
        **Then** 系统弹出提示 “该证件号码已被注册”  

        3.2.4 用户输入格式错误  
        Scenario: 输入格式错误  
        **Given** 用户在注册页面  
        **When** 用户输入错误格式的用户名、密码或证件号码  
        **Then** 系统提示：
        - 用户名/密码： “格式错误！”
        - 邮箱： “请输入有效的电子邮箱地址！”
        - 证件号码： “请正确输入18位的证件号码！”

        3.2.5 用户注册成功  
        Scenario: 正确填写所有信息并验证通过  
        **Given** 用户填写所有必填项并符合格式要求  
        **When** 用户点击“下一步”并输入正确验证码  
        **Then** 系统提示 “注册成功”  
        **And** 页面跳转至登录页
